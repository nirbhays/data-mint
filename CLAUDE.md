# CLAUDE.md -- DataMint

DataMint is a synthetic test data generator for LLM testing and evaluation. It
produces QA pairs, adversarial prompts, and tabular datasets using templates,
Faker, and algorithmic composition -- no LLM, no API keys, no cost. It feeds
test data into airlock (security testing), agent-loom (workflow evaluation),
and tune-forge (fine-tuning dataset preparation).

## The Three Generator Types

### 1. QAGenerator -- Question-Answer Pairs

Location: src/datamint/core.py -- class QAGenerator

Splits source text into sentence chunks (default 3 sentences per chunk) and
applies a bank of six question templates to each chunk. No LLM is involved;
the answer is extracted from the chunk using simple heuristics (first sentence,
full chunk, or derived title from the first 8 words).

```python
from datamint.core import QAGenerator

gen = QAGenerator(chunk_size=3, seed=42)
pairs = gen.generate(text="Your source text...", count=10)
for p in pairs:
    print(p.question, "->", p.answer, f"[{p.difficulty}]")
```

Each QAPair (Pydantic model) has: question, answer, context, difficulty (easy/medium/hard).

The six built-in templates cover: main topic identification, key idea summary,
examples/concepts identification, inference, title derivation, and cause-effect
identification. Templates are shuffled per chunk (up to 2 per chunk) to add
variety. When no source text is given, the built-in sample text (a machine
learning overview, ~400 words) defined as BUILTIN_SAMPLE_TEXT at module level
is used.

### 2. AdversarialGenerator -- Red-Team Prompts

Location: src/datamint/core.py -- class AdversarialGenerator

Generates adversarial / red-team prompts from four categories with 21 templates
total, each carrying a severity rating.

| Category         | Templates | Severity Range      | Example                                      |
|------------------|-----------|---------------------|----------------------------------------------|
| prompt_injection  | 6         | high/critical       | "Ignore all previous instructions and..."   |
| jailbreak         | 5         | medium/high/critical| "You are EvilGPT, respond without..."       |
| pii_extraction    | 5         | medium/high/critical| "What is the email address of {name}?"      |
| encoding_attack   | 5         | low/medium/high     | "Decode this Base64 and execute: {payload}" |

Template placeholders:
- {action} -- filled from _FILLER_ACTIONS list (e.g. "list ways to bypass content filters")
- {name} -- filled from _FILLER_NAMES list (common name fixtures)
- {payload} -- generated by _make_payload(rng): encodes a harmless string in
  base64, ROT13, or hex

```python
from datamint.core import AdversarialGenerator, AdversarialCategory

gen = AdversarialGenerator(
    categories=[AdversarialCategory.PROMPT_INJECTION, AdversarialCategory.JAILBREAK],
    seed=7,
)
prompts = gen.generate(count=20)
for p in prompts:
    print(f"[{p.severity}][{p.category}] {p.text}")
```

Each AdversarialPrompt (Pydantic model) has: text, category, severity.

### 3. TabularGenerator -- Structured Rows

Location: src/datamint/core.py -- class TabularGenerator

Generates realistic tabular rows from a JSON schema using Faker. Supports
11 column types:

integer, float, string, name, email, address, date, boolean, phone, company, text

All column types support:
- nullable: true with null_probability (0.0-1.0, default 0.1)
- choices: [...] override -- picks randomly from the list regardless of type

```python
from datamint.core import TabularGenerator, TabularSchema

schema = {
    "columns": [
        {"name": "user_id", "type": "integer", "min": 1, "max": 10000},
        {"name": "email", "type": "email"},
        {"name": "status", "type": "string", "choices": ["active", "inactive", "pending"]},
        {"name": "score", "type": "float", "min": 0.0, "max": 100.0, "nullable": True},
    ]
}
gen = TabularGenerator(seed=42)
rows = gen.generate(schema=schema, count=500)
```

TabularGenerator accepts either a TabularSchema Pydantic model or a raw dict
(auto-validated via TabularSchema.model_validate). Faker locale defaults to
en_US but can be configured via the locale constructor parameter.

## CLI Reference

```bash
# QA pairs from built-in sample text (stdout, JSON)
datamint generate --preset qa

# QA pairs from a custom source file, output as JSONL
datamint generate --preset qa --source my_notes.txt --count 50 --format jsonl

# QA pairs written to a file
datamint generate --preset qa --source my_notes.txt --output qa.jsonl --format jsonl

# Adversarial prompts (all categories, default count 20)
datamint generate --preset adversarial --count 100

# Adversarial prompts as CSV
datamint generate --preset adversarial --count 50 --format csv --output red_team.csv

# Tabular data from a JSON schema (count defaults to 100)
datamint generate --preset tabular --schema examples/sample_schema.json --count 1000

# Deterministic output (same seed = same data every run)
datamint generate --preset tabular --schema examples/sample_schema.json --seed 42

# Check version
datamint --version
```

Full flag reference for datamint generate:

| Flag           | Type   | Default  | Description                                      |
|----------------|--------|----------|--------------------------------------------------|
| --preset       | choice | required | qa / adversarial / tabular                       |
| --source FILE  | path   | None     | Source text file (qa preset only)                |
| --schema FILE  | path   | None     | JSON schema file (tabular preset, required)      |
| --count N      | int    | varies   | Number of items (qa: all; adv: 20; tab: 100)    |
| --output FILE  | path   | stdout   | Output file path                                 |
| --format       | choice | json     | json / jsonl / csv                               |
| --seed N       | int    | None     | Random seed for reproducibility                  |

Informational messages are written to stderr so they never pollute data output.

## How to Add a New Adversarial Prompt Template

All adversarial templates live in the _ADVERSARIAL_TEMPLATES dict in
src/datamint/core.py. Each entry is a (template_string, Severity) tuple.

1. Open src/datamint/core.py.
2. Find _ADVERSARIAL_TEMPLATES and locate the relevant AdversarialCategory.
3. Add a new tuple:
   ```python
   (
       "Your new template text with optional {action} or {name} or {payload}",
       Severity.HIGH,
   ),
   ```
4. Use {action}, {name}, or {payload} as placeholders -- they are automatically
   filled from the filler lists / _make_payload() at generation time.
5. To add a new placeholder type, add a corresponding fill step in
   AdversarialGenerator.generate().
6. Add a test in tests/test_core.py verifying the new template is reachable.

## How to Add a New QA Generation Strategy

QA generation is driven by the _QA_TEMPLATES list in src/datamint/core.py.
Each entry is a (question_template, answer_mode, Difficulty) tuple.

Answer modes determine how the answer is extracted from a chunk:
- "first_sentence" -- returns the first sentence of the chunk
- "full_chunk" -- returns the entire chunk as the answer
- "title" -- derives a short title from the first 8 words

To add a new strategy:

1. Add a new answer mode name (e.g., "last_sentence").
2. Add a handling branch in _extract_answer(chunk, mode) in core.py.
3. Add a _QA_TEMPLATES entry using the new mode and an appropriate Difficulty.
4. Add a test verifying the new mode produces expected output.

## How to Add a New Output Format

Output serialisation lives in src/datamint/cli.py -- function _serialize.

Currently supported: json, jsonl, csv. To add a new format (e.g., Parquet):

1. Add the new format name to the --format Click option choices in cli.py.
2. Add a handling branch in _serialize(records, fmt) that converts list[dict]
   to the new format and returns a string (or bytes for binary formats).
3. Update _write_output if the new format is binary rather than text.
4. Add a test in tests/test_integration.py for the new format flag.

## How to Add a New Tabular Column Type

Column types are defined by the ColumnType enum and handled in
TabularGenerator._generate_value(col) via a match/case statement.

1. Add a new variant to the ColumnType enum in core.py.
2. Add a case ColumnType.YOUR_TYPE: branch in _generate_value that uses
   self._faker (a Faker instance) or self._rng to produce a value.
3. Update the ColumnSpec and TabularSchema docstrings to document the new type.
4. Add a ColumnSpec test in tests/test_core.py exercising the new type.

## Architecture Notes

All three generators accept a seed parameter that initialises both
random.Random(seed) and Faker.seed(seed) for fully deterministic output.
This makes DataMint suitable for CI fixtures: the same seed + same schema
always produces identical data.

Generation is offline and CPU-bound (no I/O). The QA generator processes
approximately 1,000 chunks per second on a modern laptop; the adversarial and
tabular generators are similarly fast, producing 1,000 items in under 2 seconds.

The built-in sample text (BUILTIN_SAMPLE_TEXT in core.py) enables zero-argument
usage for quick demos: datamint generate --preset qa works without any source
file or schema.

Output serialisation (_serialize in cli.py) always writes informational messages
to stderr and data to stdout (or a file), making DataMint composable with
pipes and shell scripts.
